# 自我帮助小助手：代码上交报告

## 1. 项目简介

本项目是一个面向研究生日常工作流的本地小助手，主要解决两个问题：

1. 桌面和文件夹资料过多时，快速根据文件名或关键词查找文件，并展示文件路径和大致内容。
2. 面对大型代码项目时，自动生成包/模块级结构说明，并在选择某个包后展示其中函数的作用。

项目默认本地运行，文件查找和离线代码分析不需要上传资料；如果需要更自然的函数解释，可以在 `.env` 中配置豆包、DeepSeek、ChatGPT 或 Gemini 的 API Key 进行模型增强。

## 2. 核心功能

### 文件查找

- 支持输入文件名片段或内容关键词，例如 `leetcode`、`组会`、`论文`。
- 搜索范围可以留空，默认扫描桌面、文档和当前工具目录。
- 返回匹配文件列表。
- 点击文件后展示完整路径和内容摘要。
- 自动生成 Markdown 报告和 JSON 索引。

### 代码分析

- 输入代码项目路径，例如 `F:\Desktop\xla`。
- 自动扫描代码文件，识别函数、行号、参数、调用线索和函数作用。
- 先展示包/模块列表。
- 点击某个包后，展示该包下每个函数的大致用途。
- 支持离线分析和模型增强分析。

## 3. 技术实现

项目使用 Python + 原生 HTML/CSS/JavaScript 实现。

- `local_server.py`：本地 HTTP 服务，提供页面访问和 `/api/analyze` 分析接口。
- `self_help_assistant.py`：文件扫描、关键词查找、文件分类、摘要提取和报告生成。
- `code_analyzer.py`：代码项目函数级分析，支持离线规则分析和可选模型增强。
- `code_analysis_text_summary.py`：根据代码分析索引生成文字版代码总结。
- `visualize_assets.py`：生成文件查找和资料扫描相关图片。
- `visualize_code_analysis.py`：生成代码分析相关图片。
- `3D桌面展示.html`：主交互页面，包含文件查找和代码分析两个工作流。

## 4. 运行方式

进入项目目录：

```powershell
cd "F:\user_documenbts\New project\self_help_assistant"
```

启动本地服务：

```powershell
python .\local_server.py --port 8765
```

浏览器打开：

```text
http://127.0.0.1:8765/
```

## 5. 配置说明

`.env` 用于配置模型 API Key，不应上传到 GitHub。

仓库中只上传 `.env.example`，用户复制为 `.env` 后自行填写：

```env
MODEL_PROVIDER=doubao
DOUBAO_API_KEY=
DEEPSEEK_API_KEY=
CHATGPT_API_KEY=
GEMINI_API_KEY=
```

不配置 API Key 时，代码分析仍可使用离线模式。
